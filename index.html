Product Requirements Document (PRD)

Vision, user roles, features, and success metrics

Vision & Mission

PlacePro is an AI-powered Learning Management System (LMS) built specifically for placement preparation. It combines gamified quizzes, AI mock interviews, real-time job aggregation from the internet, personalised career roadmaps, a code challenge module, and a resume builder — all in one platform. The platform targets students at every education stage: post-Class 10, post-Intermediate, diploma holders, and graduates.

💡
Tagline: "From classroom to career — every step, gamified." The core differentiator is the complete loop: learn → practice → interview → apply → track progress — all without leaving the platform.

Problem Statement

Fragmented tools

Job boards, resume tools, interview prep, and learning are spread across dozens of apps. Students lose momentum switching between them.

No personalised roadmap

A Class 10 pass-out or diploma holder has no structured guide from their current state to their goal career.

Passive interview prep

PDFs and YouTube don't simulate real interviews. No feedback loop on spoken answers, tone, or confidence.

Motivation collapse

Without streaks, XP, leaderboards, and rewards, students abandon prep platforms within weeks.

User Roles

Student

Any registered learner. Education level can be Class 10 pass, Intermediate, Diploma, or Graduate.

  • ✓ Take topic-based quizzes generated from admin syllabus
  • ✓ Attempt monthly admin-scheduled quizzes
  • ✓ AI mock interview (camera + mic + proctoring) or manual interview
  • ✓ Browse and apply to live job listings (India + selected countries)
  • ✓ View and follow AI-generated career roadmap
  • ✓ Build resume and get live ATS score + feedback
  • ✓ Solve coding challenges graded by AI
  • ✓ Track XP, streaks, badges, level, and leaderboard rank
  • ✓ View project showcase on landing page

Admin

Platform operators who manage all content, assessments, and configuration.

  • ✓ Manage syllabus topics — upload PDF/text per topic (updatable anytime)
  • ✓ Create and publish monthly quizzes with AI-assisted question generation
  • ✓ Upload project showcase (images, GitHub links, deployed URLs)
  • ✓ Manage user accounts, view platform analytics
  • ✓ Configure ad placements for revenue
  • ✓ Review flagged interview sessions
  • ✓ Set career roles and approve roadmap content

Landing Page Visitor

Unauthenticated visitor. Can view the landing page, project showcase, feature overview. Must sign up or log in to access any learning feature.

Supabase Trigger: auto-create profile on signup

-- Run in Supabase SQL Editor
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, full_name, role)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'full_name', ''),
    'student'
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION handle_new_user();

Leaderboard Logic

Leaderboard is a materialised view refreshed by a cron job (pg-boss, every Monday 00:00 UTC for weekly, 1st of month for monthly). The leaderboard_snapshots table stores the frozen rank per period. Live leaderboard (during the week) is computed from xp_transactions filtered by created_at >= start_of_period using a SQL window function.

-- Live weekly leaderboard query
SELECT
  p.id,
  p.full_name,
  p.avatar_url,
  p.level,
  SUM(x.amount) AS xp_this_week,
  RANK() OVER (ORDER BY SUM(x.amount) DESC) AS rank
FROM xp_transactions x
JOIN profiles p ON p.id = x.user_id
WHERE x.created_at >= date_trunc('week', now())
GROUP BY p.id, p.full_name, p.avatar_url, p.level
ORDER BY xp_this_week DESC
LIMIT 100;

Module 1 — Quiz Question Generation

Endpoint: POST /api/quiz/generate

System prompt
You are a quiz question generator for an educational platform.
Generate exactly {question_count} multiple-choice questions based on the provided syllabus.
Respond ONLY with a valid JSON array. No preamble, no markdown, no explanation outside the JSON.

Format:
[
  {
    "question": "Question text here",
    "options": ["Option A", "Option B", "Option C", "Option D"],
    "correct_index": 1,
    "explanation": "Why this answer is correct, in 1-2 sentences."
  }
]

Rules:
- Difficulty: {difficulty} (beginner|intermediate|advanced)
- All questions must come from the provided syllabus only
- Correct answers should be evenly distributed across A, B, C, D
- Explanations must be educational and reference the syllabus concept
- Do not repeat similar questions
User message
Syllabus content for topic "{topic_name}":
---
{syllabus_text}
---
Generate {question_count} {difficulty} questions.

Server-side security rule

After generation, store questions in quiz_questions table. The correct_option_index column is excluded from all frontend API responses. Validation happens server-side at POST /api/quiz/submit.

Module 2 — AI Interview

Flow

1
Session start

POST /api/interview/start → creates interview_sessions row, returns session ID and first AI question as text + base64 audio (ElevenLabs TTS).

2
Student answers

Browser records mic audio, sends to POST /api/interview/answer as audio blob. Server transcribes via Whisper API, appends to transcript, generates next question via Claude.

3
Proctoring

Browser sends proctor events to POST /api/interview/proctor-event (tab switch, face absent, mic muted). 3 warnings → server sends terminate signal via Supabase Realtime.

4
Session end

POST /api/interview/complete → Claude evaluates full transcript, returns structured feedback JSON. XP credited. Status set to 'completed'.

Evaluation prompt (on session complete)
You are evaluating a mock job interview for a {topic_name} role.

Interview transcript:
{transcript_json}

Score the candidate on three dimensions, each 0-100:

Respond ONLY with valid JSON:

  "communication": 72,

Question N / Total. Timer countdown (red when <30s). 4 option buttons. No back navigation. Tab switch detection active.

Immediate green/red flash on selected option. AI explanation shown below correct answer. "Next" button. XP animation on correct.

Step 4 — Results

Score circle (e.g. 72%). XP earned badge. Per-question breakdown. "Share to leaderboard" button. "Retry" and "New quiz" CTAs.
